# Project Structure

This project is a Java 21 Spring Boot application for secure file upload, download, and token-based sharing. It follows a conventional layered structure (controller -> service -> repository) with domain models, security configuration, and a small static frontend.

## High-Level Layout

- pom.xml: Maven build and dependency configuration.
- src/main/java/com/cyber/securefilesharing: Application code.
- src/main/resources: Spring Boot configuration and static web assets.
- target/: Maven build output (generated).

## Java Packages

- config/: Security and JWT setup.
  - JwtAuthenticationFilter: Extracts and validates JWTs for protected endpoints.
  - JwtService: Token creation and validation utilities.
  - SecurityConfig: Spring Security rules and filter wiring.
- controller/: HTTP API endpoints.
  - AuthController: Registration and login endpoints.
  - FileController: Upload, list, download, and delete file endpoints.
  - FolderController: Folder management endpoints.
  - ShareController: Share link creation and token download endpoints.
- model/: Domain entities.
  - UserAccount, FileMetadata, Folder, FolderTemplate, ShareToken.
- repository/: Spring Data JPA repositories.
  - UserRepository, FileMetadataRepository, FolderRepository, FolderTemplateRepository, ShareTokenRepository.
- service/: Business logic and storage handling.
  - UserService: User creation and lookup logic.
  - FolderService: Folder operations.
  - FileStorageService: Disk storage and file metadata coordination.

## Resources

- src/main/resources/application.properties: Application configuration.
- src/main/resources/static/index.html: Minimal UI entry point.
- src/main/resources/static/css/styles.css: Frontend styles.
- src/main/resources/static/js/app.js: Frontend behavior and API calls.

## Request Flow (Typical)

1. Request hits a controller endpoint.
2. Controller delegates to a service for business logic.
3. Service uses repositories for data access and FileStorageService for disk I/O.
4. SecurityConfig and JwtAuthenticationFilter enforce authentication where needed.

## Notes

- Files are stored on disk under storage/ (created at runtime).
- target/ is generated by Maven and can be cleaned with mvn clean.
