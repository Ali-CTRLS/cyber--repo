SLIDE 5 — SECURITY ARCHITECTURE (CY FOCUS)
===============================================================
Layout: Left column (security layers) + Right column (CIAAA badges)

SLIDE HEADING:
  "🔐 Security Architecture"
  Font: Poppins Bold, 30px, #1e40af

LEFT COLUMN — SECURITY LAYERS (stacked blue-bordered rows):

  [🔑] Password Hashing
       Werkzeug generate_password_hash — PBKDF2 default
       Stored as hash; never plaintext

  [🍪] Session-Based Auth
       Flask-Login · @login_required decorator
       Role-aware redirect (_redirect_by_role)

  [🛡️] CSRF Protection
       Flask-WTF on all POST forms
       form.hidden_tag() in every template

  [🔒] AES-256-GCM File Encryption
       cryptography.hazmat AESGCM · 12-byte random nonce
       Nonce + ciphertext stored in *.enc file

  [🚦] RBAC Enforcement
       role check (current_user.role != "doctor/patient") → abort(403)
       Report access: patient_id ownership verified before serving

RIGHT COLUMN — CIAAA BADGES (rounded pill labels, blue fill):

  ● Confidentiality  → AES-256-GCM encryption + RBAC
  ● Integrity        → GCM authentication tag (tampering = decrypt failure)
  ● Availability     → Flask dev server + SQLite always accessible
  ● Authentication   → Flask-Login + Werkzeug password hashing
  ● Authorization    → Role checks + ownership guards on all routes

  ↓ arrow label: "key → DB · ciphertext → disk"

STAGE 3 — STORE (blue card):
  Icon: 💾
  Label: "Stored Separately"
  Sub: "Key (base64) → Report.encryption_key in SQLite"
       "Encrypted file → storage/reports/{uuid}.enc"

DOWNLOAD REVERSE FLOW (below, dashed arrow ← direction):
  Patient Requests → Read *.enc from disk
  → decrypt_file(encrypted_data, key_b64)
  → nonce = data[:12] · ciphertext = data[12:]
  → AESGCM.decrypt → raw bytes served via send_file()

CALLOUT BOX (light blue #dbeafe):
  "⚠️ GCM Integrity Tag: Any tampering with ciphertext causes
   decryption to FAIL — no silent data corruption"

SLIDE 7 — EDGE CASES & HANDLING
===============================================================
Layout: Two-column table card

SLIDE HEADING:
  "⚠️ Edge Cases & Error Handling"
  Font: Poppins Bold, 30px, #1e40af

TABLE (white background, blue header row, alternating light-gray rows):

  | Scenario                           | Handling                              |
  |------------------------------------|---------------------------------------|
  | Invalid login credentials          | flash("Invalid username or password") |
  | Unauthorized report access         | abort(403) — HTTP 403 Forbidden       |
  | Report before CONFIRMED status     | flash warning → redirect detail page  |
  | Report already exists              | flash warning → no duplicate created  |
  | No file attached to report         | flash warning on download attempt     |
  | Cancel already-cancelled appt      | flash warning — no state change       |
  | Confirm non-PENDING appointment    | flash warning — guard in confirm route|
  | Wrong role accesses doctor route   | abort(403) enforced on every route    |

NOTE BOX (bottom, light blue):
  "All guards implemented via explicit if-checks before DB mutations.
   No silent failures — every edge case surfaces a user-visible message."
